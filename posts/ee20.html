搭建部分

安装 hexo

此处略去 Git 和 Node 安装教程

在目标路径打开 CMD 窗口，输入 npm install -g hexo-cli 安装 Hexo 环境。

安装完后输入hexo -v验证是否安装成功。

初始化 hexo 项目 ，即初始化我们的网站项目 ，输入 hexo init 初始化文件夹。

然后进入 Demo ，接着输入 npm i 安装相关依赖。

  • 新建完成后，文件夹 Demo 目录下有：

    • node_modules: 依赖包

    • public：存放生成的页面

    • scaffolds：生成文章的一些模板

    • source：用来存放你的文章

    • themes：主题

    • _config.yml: 博客的配置文件

  • 这样本地的网站配置就弄好啦，输入hexo generate生成静态网页

  • 然后输入hexo s -p 8000（指定端口）打开本地服务器，然后浏览器打开 http://localhost:8000/。

添加说说页面

文档 Artitalk.js

  1. 前往 LeanCloud 国际版，注册账号，根据 LeanCloud 的提示绑定手机号和邮箱。
  2. 绑定完成之后点击创建应用，应用名称随意，接着在结构化数据中创建 class，命名为 shuoshuo。
  3. 在你新建的应用中找到结构化数据下的用户。点击添加用户，输入想用的用户名及密码。
  4. 回到结构化数据中，点击 class 下的 shuoshuo。找到权限，在 Class 访问权限中将 add_fields 以及 create 权限设置为指定用户，输入你刚才输入的用户名会自动匹配。为了安全起见，将 delete 和 update 也设置为跟它们一样的权限。
  5. 然后新建一个名为atComment的 class，权限什么的使用默认的即可。
  6. 点击 class 下的 _User 添加列，列名称为 img，默认值填上你这个账号想要用的发布说说的头像 url，这一项不进行配置，说说头像会显示为默认头像 —— Artitalk 的 logo。
  7. 在最菜单栏中找到设置-> 应用 keys，记下来 AppID 和 AppKey ，一会会用。
  8. 最后将 _User 中的权限全部调为指定用户，或者数据创建者，为了保证不被篡改用户数据以达到强制发布说说。

添加友链朋友圈

文档：hexo-filter-fcircle

原理：github action 爬虫抓取友链文章并作基本处理，推送至 leancloud → vercel 获取 leancloud 数据后进行二次处理后生成 api

  1. fork 仓库：fork 友链朋友圈的项目仓库 https://github.com/Rock-Candy-Tea/hexo-circle-of-friends

  2. 配置环境变量：点击 fork 后的仓库的 Settings -> Secrets -> New repository secret

  3. 启用 github action 点： 击 fork 后的仓库的 Actions -> I understand my workflows, go ahead and enable them

  4. 启用 workflow: 点击 update-friends-posts 并启用 workflow

  5. 之后点击仓库右上角 star 来完成第一次程序运行（除非你不是已经适配的主题，在配置项中会有相应解决办法）

  6. 这时你应该能在 leancloud 上的 friend_list friend_poor 看到上传的数据

  7. 此后 workflow 会在每天的 8,12,17,22 点各会有两次自动更新爬取。
